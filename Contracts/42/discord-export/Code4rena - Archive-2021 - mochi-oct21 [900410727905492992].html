@terMAX It looks like the two tests above aren't correct (or at least, they don't match with the MochiVault contract within the repo). My observations I've found so far: - The only check that results in the ">cf" error message is within the withdraw() function, and from what I can tell, withdraw() is not called from within the borrow() function, which is the function under test. (And conceptually, calling withdraw() within borrow() doesn't make much sense). - Based on what I can see in the withdraw() function, if a collateral factor check were intended for the borrow() function, I would assume the check would need to occur after calculating the new debt (details[_id].debt + _amount). There is a check for minimum debt, but nothing for collateral factor. - Line 228 of MochiVault.sol appears to squash the ability to borrow more than your collateral factor (https://github.com/code-423n4/2021-10-mochi/blob/main/projects/mochi-core/contracts/vault/MochiVault.sol#L228). Compare that line with the >cf check within the withdraw() function (https://github.com/code-423n4/2021-10-mochi/blob/main/projects/mochi-core/contracts/vault/MochiVault.sol#L204), and you'll see that the new debt (_amount) can never be greater than the collateral factor, since line 228 readjusts _amount such that it's always less than or equal to maxMinted. Given the above, perhaps this is why the tests fail? Looks like the borrow() function is not supposed to revert with >cf, so there may be an issue with the tests. (edited)
